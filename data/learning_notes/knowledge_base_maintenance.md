# local-rag-assistant 知识库补充与维护

> 说明：本文档用于支持“资料不足时如何补充知识库”的项目使用指导类问答。

## 资料目录结构

local-rag-assistant 的资料主要分为两类：

- `data/`：公开示例资料，用于演示行业、岗位、项目、学习笔记和示例简历。
- `private_data/`：本地私有资料，用于保存用户自己的简历、岗位 JD、项目资料和其他不应提交到 GitHub 的内容。

项目中的 `.gitignore` 应避免提交 `.env`、`private_data/` 和 `qdrant_storage/`，防止 API Key、真实简历和本地向量库被误提交。

## 适合补充的资料类型

如果模型回答“资料不足”，通常不是只需要多上传文件，而是需要补充更可回答的事实资料。

适合补充的资料包括：

- 简历事实：技能、项目经历、教育经历、实习经历、可证明成果。
- 岗位事实：岗位定位、主要职责、能力要求、加分项、不重点要求。
- 项目事实：项目定位、解决的问题、核心功能、技术栈、RAG 流程、局限性、简历表达、面试表达。
- 学习笔记：概念解释、框架对比、工具差异、常见面试问法。
- 行业资料：业务场景、数据特点、常见 AI 应用落地点。

## 推荐的事实卡片结构

补充项目资料时，建议使用事实卡片，而不是只写长篇流水账。

推荐结构：

```text
# 项目名称

## 项目定位

## 解决的问题

## 核心功能

## 技术栈

## RAG 或 AI 流程

## 适合写进简历的表达

## 面试时可以这样说

## 局限性

## 后续改进方向
```

补充岗位资料时，建议使用：

```text
# 岗位名称

## 岗位定位

## 主要职责

## 能力要求

## 加分项

## 不重点要求
```

## 修改资料后的索引更新

补充或修改资料后，必须重新构建索引，否则向量库里仍然是旧内容。

命令行入口：

```powershell
python src/build_index.py
```

在当前学习环境中，也可以使用共享虚拟环境执行：

```powershell
..\cli-ai-assistant\.venv\Scripts\python.exe src\build_index.py
```

Web 工作台入口：

```text
索引状态 -> 更新索引
```

后端 API 入口：

```text
POST /api/index/build
```

## 验证方法

更新索引后，应使用固定问题验证资料是否生效。

推荐验证方式：

- 问“某项目解决了什么问题”，看是否命中项目事实卡片。
- 问“某项目局限性是什么”，看是否引用局限性片段。
- 问“这份简历应该怎么改”，看是否同时命中当前简历、目标岗位和项目资料。
- 问“资料不足时怎么补充知识库”，看是否能说出资料目录、事实卡片和索引更新步骤。

## 常见错误

- 只上传长文档，但没有清晰标题，导致切分后的片段不容易命中。
- 资料里没有直接回答问题的事实，只能让模型猜。
- 修改资料后忘记更新索引。
- 把真实隐私资料放入公开示例目录或提交到 GitHub。
- 认为本地 Qdrant 等于完全隐私安全，但 Embedding 和 Chat 模型调用仍可能把文本发送到第三方模型 API。
