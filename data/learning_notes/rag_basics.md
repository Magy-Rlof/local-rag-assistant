# 示例学习笔记：RAG 基础

> 说明：本文档为模拟示例数据，仅用于演示 RAG 检索流程。

## RAG 定义

RAG 是 Retrieval-Augmented Generation 的缩写，核心流程是先检索相关资料，再让模型基于资料生成回答。

## 文档加载

文档加载是把本地或远程资料读入程序，例如 Markdown、PDF、Word 或网页内容。学习阶段可以先从 Markdown 开始。

## 文本切分

文本切分是把长文档拆成较小片段。按标题切分适合结构清楚的 Markdown 文档。

## Embedding

Embedding 是把文本转换成向量，用于计算语义相似度。它比简单关键词匹配更适合处理问法不同但意思接近的问题。

## 向量数据库

向量数据库用于保存文档片段向量，并根据用户问题向量检索最相似的片段。Qdrant、Milvus、Chroma 和 pgvector 都是常见选择。

## 引用来源

RAG 系统应返回引用来源，帮助用户判断回答依据。资料不足时，系统应明确说明无法确定。
