# 示例学习笔记：Tool Calling

> 说明：本文档为模拟示例数据，仅用于演示 RAG 检索流程。

## 基本概念

Tool Calling 的核心是让模型判断是否需要外部工具，真正执行工具的是程序。模型负责理解意图和组织语言，程序负责调用函数、数据库或外部接口。

## 常见工具

- 文件读取
- 数据库查询
- 搜索接口
- 计算函数
- 业务系统 API

## 与 RAG 的区别

RAG 主要用于从文档中检索相关片段，再让模型基于资料回答。Tool Calling 更适合调用明确的程序能力，例如查订单、读取项目配置或执行计算。

## 求职表达

在项目中可以说明：模型不直接操作外部系统，而是由后端程序根据模型输出或用户意图调用工具，并对结果做安全处理。
